Everything in one place

One world. Every tool. Infinite stories.

Mimir is a desktop application for structured worldbuilding. Your entries live as plain markdown files with YAML frontmatter — readable in any editor, ready for AI, and native to any game engine. 13+ built-in entry types. Custom schemas. Five relationship trees. Interactive maps. A full quest and dialogue graph editor with simulation mode. Available on Windows, macOS, and Linux.

Structured Entries

13+ built-in entry types — characters, species, cultures, places, factions, artifacts, events, languages, story arcs, quests, dialogues, maps, and more. Create unlimited custom entry types via the schema editor with 9 field types including text, dates, links, link lists, tags, and relationship lists. Form-based editing with focus mode for distraction-free work.

Quest & Dialogue Editor

A node-based quest and dialogue graph editor built on React Flow. 8 node types including dialogue lines, branching choices, conditions, consequences, and subgraph references. A typed variable system with 4 scopes (global, per-character, per-faction, per-place). Full simulation mode with step debugger, state inspector, coverage tracking, and replay logging. Direct engine export to Unreal, Unity, and Godot.

Ask Mimir AI

Ask Mimir is the in-app AI assistant that knows your world. Built on a strict-honesty invariant — it answers only from your lorebible content and confesses ignorance with stub-creation suggestions. Three quality tiers (Fast, Quality, Ultimate). Per-entry focused conversations. Prompt caching for token savings. Pay-as-you-go credits on the free tier, monthly quota on Studio.

AI & Engine Integration

Mimir's MCP server (Model Context Protocol — an open standard for connecting AI tools to your data) gives Claude, Cursor, and other AI tools direct access to your world data. Export to Unreal Engine DataTable CSV and generic JSON at V1, with Unity ScriptableObject and Godot Resource exports coming in V1.5.

Custom Schemas

No developer skills needed. Create unlimited custom entry types with 9 field types. Five relationship trees (family, genus, place, culture, faction). Interactive maps with pin linking. Custom calendars, timelines, and story arcs. Wiki-links with autocomplete and backlinks. Git auto-commit and version history. Naming consistency engine.

Translator

A full constructed-language workbench. Define registers, cultural bias, grammar rules, naming conventions, pronoun hierarchy, and vocabulary per language. Translate English text into your invented language — consistently, every time. AI captures new vocabulary automatically. Per-word regenerate controls, pronunciation popover, 500-entry history. Calibration phrases let you set the exact tone.

How it works

Four steps to a living world.

  1. i

    Pick a folder.

    Mimir reads from a folder you choose. No proprietary database, no upload step. Your world lives on your machine as plain markdown files.

  2. ii

    Write entries.

    Use the structured editor or write markdown directly. Either way, the same plain files end up on disk — readable in any editor, ready for AI.

  3. iii

    Link everything.

    Reference a character, a faction, a place. The links light up the timeline, the family tree, and Ask Mimir.

  4. iv

    Ship to your engine.

    Export to Unreal, Unity, or Godot. Or point Claude Projects, Cursor, or ChatGPT at the folder. Your tools learn your world.
