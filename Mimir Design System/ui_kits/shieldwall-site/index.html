Chapter ii — how we work

Slow over snappy. Crafted over clever.

We build the things we'd want as players. No live-service treadmill. No telemetry-driven design loops. We make games and software that respect the time you spend with them — and stop asking for it when they're done.

Mimir came out of frustration: every world we tried to build for our own tabletop campaigns drifted across Word docs, wikis, and engine data files until it stopped agreeing with itself. We needed a single source of truth. So we made one.

Ashborn came out of love: for sagas, for the long winter, for the kind of game where the table goes quiet when a piece moves.
